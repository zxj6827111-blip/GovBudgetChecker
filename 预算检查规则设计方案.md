# 预算公开报告检查规则设计方案

> **编制日期**：2026-02-24  
> **参考样本**：  
> - `X/上海市普陀区人民政府办公室25年预算.pdf`（部门预算，22页）  
> - `X/上海市普陀区人民政府办公室单位25年预算.pdf`（单位预算，21页）  
> **对标规则**：`rules/v3_3.yaml` 中的决算检查规则（D-001 ~ D-009、R001 ~ R014）

---

## 一、预算报告结构分析

### 1.1 部门预算 vs 单位预算 — 结构对比

| 章节 | 部门预算 | 单位预算 | 差异说明 |
|------|---------|---------|---------|
| 封面 | "区级**部门**预算" / "预算**主管部门**：XXX" | "区级**单位**预算" / "预算**单位**：XXX" | 封面称谓体系不同 |
| 一、主要职能 | "**部门**主要职能" | "**单位**主要职能" | 内容一致 |
| 二、机构设置 | 列出下属单位数量和清单（如"行政单位1家，事业单位1家"） | 列出内设机构（如"设10个内设机构"） | **部门需声明下属单位覆盖关系** |
| 三、名词解释 | 八个标准名词 | 八个标准名词 | 内容一致 |
| 四、预算编制说明 | 收支概况 + 12条逐科目明细 | 收支概况 + 12条逐科目明细 | 内容一致，金额一致（部门=单位的情况） |
| 五、预算表（9张） | 表头均为"编制**部门**" | 表头均为"编制**单位**" | 表头称谓不同 |
| 六、其他相关情况说明 | 五大子章节 | 五大子章节 | 内容基本一致 |
| **七、项目经费情况说明** | **有**（包含7个标准子项） | **无** | **部门预算独有** |

### 1.2 九张预算表清单

| 表序 | 表名 | 单位 | 表结构要点 |
|------|------|------|-----------|
| 表1 | 财务收支预算总表 | **元** | 左侧收入（4类）、右侧支出（按功能大类），底行：收入总计=支出总计 |
| 表2 | 收入预算总表 | **元** | 按功能分类科目编码（类-款-项）展开，列含：合计、财政拨款、事业收入、事业单位经营收入、其他收入 |
| 表3 | 支出预算总表 | **元** | 按功能分类编码展开，列含：合计、基本支出、项目支出 |
| 表4 | 财政拨款收支预算总表 | **元** | 左侧拨款收入（一般公共/政府性基金/国有资本），右侧拨款支出（按功能大类，分三列） |
| 表5 | 一般公共预算支出功能分类预算表 | **元** | 同表3结构，仅限一般公共预算口径 |
| 表6 | 政府性基金预算支出功能分类预算表 | **元** | 同表5结构，**大概率为空表** |
| 表7 | 国有资本经营预算支出功能分类预算表 | **元** | 同表5结构，**大概率为空表** |
| 表8 | 一般公共预算基本支出经济分类预算表 | **元** | 按经济分类科目展开，列含：合计、人员经费、公用经费 |
| 表9 | "三公"经费和机关运行经费预算表 | **万元** | 一行表：合计、因公出国境费、公务用车（小计/购置/运行）、公务接待费、机关运行经费 |

> ⚡ **关键发现**：表1~表8的单位为"**元**"（精确到分），表9和编制说明正文的单位为"**万元**"（保留两位小数）。同一PDF内混用两种计量单位，是预算检查引擎的核心前置处理难点。

### 1.3 "其他相关情况说明"五大必备子章节

1. **三公经费预算情况说明** — 含因公出国、公务用车、公务接待三项的金额和同比说明
2. **机关运行经费预算** — 含机关和事业单位数量、机关运行经费总额
3. **政府采购预算情况** — 含采购总额及货物/工程/服务分项，中小企业预留额度
4. **绩效目标设置情况** — 含编报项目数量和涉及资金总额
5. **国有资产占有使用情况** — 含车辆数量（按用途分类）和大型设备数量

## 二、完整规则矩阵

### 2.1 表内勾稽规则（Table Internal Checks）

#### B-001：总表收支平衡
- **严重度**：`critical`
- **适用范围**：部门 + 单位
- **校验逻辑**：表1《财务收支预算总表》的 `收入总计 == 支出总计`
- **样本验证**：`91,016,775.35 == 91,016,775.35` ✅
- **容差**：`±0.01元`（即绝对精确匹配）

#### B-002：财政拨款三类分解验证
- **严重度**：`critical`
- **适用范围**：部门 + 单位
- **校验逻辑**：表4《财政拨款收支预算总表》的 `支出总计 == 一般公共预算支出 + 政府性基金预算支出 + 国有资本经营预算支出`
- **同时校验**：表4支出总计 == 表5合计 + 表6合计 + 表7合计
- **样本验证**：`91,016,775.35 == 91,016,775.35 + 0 + 0` ✅

#### B-003：基本支出人员公用拆分验证
- **严重度**：`critical`
- **适用范围**：部门 + 单位
- **校验逻辑**：表8《经济分类预算表》的 `合计 == 人员经费 + 公用经费`
- **样本验证**：`24,606,775.35 == 22,473,997.81 + 2,132,777.54` ✅
- **逐行校验**：每一行的经济分类科目也应满足 `合计 = 人员经费 + 公用经费`

#### B-004：三公经费合计=三项之和
- **严重度**：`high`
- **适用范围**：部门 + 单位
- **校验逻辑**：表9的 `合计 == 因公出国(境)费 + 公务用车购置及运行费（小计）+ 公务接待费`
- **特殊处理**：表中空白单元格视为 `0`
- **样本验证**：`867.50 == 839.00 + 0 + 28.50` ✅

#### B-007：支出总表行级汇总验证
- **严重度**：`critical`
- **适用范围**：部门 + 单位
- **校验逻辑**：表3《支出预算总表》的每一行 `合计 == 基本支出 + 项目支出`；合计行也应等于所有"类"级行之和
- **样本验证**：`91,016,775.35 == 24,606,775.35 + 66,410,000.00` ✅

### 2.2 表文勾稽规则（Table-to-Text Cross Checks）

#### B-008：编制说明科目金额↔表格交叉验证
- **严重度**：`high`
- **适用范围**：部门 + 单位
- **校验逻辑**：
  1. 提取"预算编制说明"中所有 `"XXX-YYY-ZZZ"科目 N 万元` 格式的条目
  2. 将编制说明中的万元金额 × 10000，与表2/表5的对应科目行的元金额做比对
  3. 容差：`±100元`（即 ±0.01万元，因编制说明保留两位小数万元）
- **示例**：
  - 说明：`"行政运行"科目 1,202.95万元` → 1,202.95 × 10000 = 12,029,500
  - 表5：`201 03 01 行政运行 12,029,501.84`
  - 差值：1.84元 → 在容差范围内 ✅

#### B-011：机关运行经费↔表8公用经费勾稽
- **严重度**：`high`
- **适用范围**：部门 + 单位
- **校验逻辑**：提取"其他说明"中 `"机关运行经费预算为 N 万元"` 的 N 值，与表8合计行的"公用经费"列做比对
- **单位转换**：说明中为万元，表8为元 → `N × 10000 ≈ 表8.公用经费合计`
- **容差**：`±100元`
- **样本验证**：`213.28 × 10000 = 2,132,800` vs 表8公用经费合计 `2,132,777.54` → 差值 22.46元 ✅

#### B-TbT-三公经费说明↔表9验证
- **严重度**：`high`
- **适用范围**：部门 + 单位
- **校验逻辑**：
  1. 提取"三公经费预算情况说明"中分项金额（万元单位，与表9同口径）
  2. 逐项比对：因公出国（境）费、公务用车购置及运行费（含购置费+运行费）、公务接待费
  3. 同时校验合计
- **同比说明检查**：若文字中出现"增加/减少"字样，必须同时出现原因说明文字
- **样本发现**：说明里所有项均为"与2024年预算持平"，无需额外原因说明 ✅

#### B-012：政府采购预算内部一致性
- **严重度**：`medium`
- **适用范围**：部门 + 单位
- **校验逻辑**：
  - `采购总额 == 货物 + 工程 + 服务` 子项之和
  - `中小企业预留额度 ≤ 采购总额`
- **样本验证**：`34 == 0 + 0 + 34` ✅，`34 ≤ 34` ✅

#### B-013：绩效预算资金 ≤ 项目支出
- **严重度**：`medium`
- **适用范围**：部门 + 单位
- **校验逻辑**：绩效目标涉及的项目预算资金不应超过表3的项目支出总额
- **样本验证**：`6626万元 < 6641万元（66,410,000元 ÷ 10000）` ✅
- **注意**：如果绩效资金 > 项目支出总额，则判为 `确定` 不合理

### 2.3 格式与内容规范检查

#### B-005：部门/单位称谓一致性
- **严重度**：`high`
- **适用范围**：部门 + 单位
- **校验逻辑**：
  - 部门预算：封面含"部门预算"→ 全文表头应为"编制**部门**"，不应出现"编制单位"
  - 单位预算：封面含"单位预算"→ 全文表头应为"编制**单位**"，不应出现"编制部门"
  - 正文章节标题也需对应（"部门主要职能" vs "单位主要职能"）
- **拦截级别**：出现混用 → `high`

#### B-006：下属单位覆盖性声明（仅部门预算）
- **严重度**：`medium`
- **仅适用**：部门预算
- **校验逻辑**：
  - "机构设置"章节必须声明：行政单位 N 家、事业单位 M 家
  - 必须列出所有下属单位名称
  - 必须说明哪些单位"单独编报"，哪些"并入本部"
- **样本验证**：`"行政单位1家，事业单位1家…下属事业单位上海市普陀区交流服务中心不单列预算，并入本部"` ✅

#### B-009：其他相关情况说明五大必备子章节
- **严重度**：`critical`
- **适用范围**：部门 + 单位
- **必备章节**：
  1. `三公经费预算情况说明` — 需含三项分列金额 + 同比说明
  2. `机关运行经费预算` — 需含机关数量 + 经费总额
  3. `政府采购预算情况` — 需含采购总额 + 三项分列 + 中小企业预留
  4. `绩效目标设置情况` — 需含项目数量 + 资金总额
  5. `国有资产占有使用情况` — 需含车辆数量（按用途分类）+ 大型设备数量

#### B-010：项目经费情况说明结构检查（仅部门预算）
- **严重度**：`medium`
- **仅适用**：部门预算
- **校验逻辑**：如果存在项目支出，则必须有"项目经费情况说明"章节
- **标准子项结构**（共7项）：
  1. 项目概述
  2. 立项依据
  3. 实施主体
  4. 实施方案
  5. 实施周期
  6. 年度预算安排
  7. 绩效目标

#### B-014：空表声明检查
- **严重度**：`critical`
- **适用范围**：部门 + 单位
- **校验逻辑**：
  - 若表6（政府性基金）全零或无数据行，必须有注释：`"本部门/单位2025年无政府性基金预算财政拨款安排的预算，故本表为空表"`
  - 若表7（国有资本经营）全零或无数据行，必须有注释：`"本部门/单位2025年无国有资本经营预算财政拨款安排的预算，故本表为空表"`
- **样本验证**：两份PDF均有对应注释 ✅

#### B-015：占位符扫描
- **严重度**：`critical`
- **适用范围**：部门 + 单位
- **扫描模式**：
  ```
  XX、××、XXX、{单位}、{部门}、某某、待填、待定、X年、○○
  ```
- **命中即报错**：任何占位符残留都表明报告未完成编辑

#### B-016：年份一致性检查
- **严重度**：`high`
- **适用范围**：部门 + 单位
- **校验逻辑**：
  - 封面、目录、全部表头年份应统一为"2025年"
  - 编制说明中的同比参照年份应为"2024年"
  - 不应出现非相邻年份（如2023、2026等），除非在特定上下文（如立项依据引用）
- **白名单**：`"截至2024年8月31日"` 这类资产截止日期允许出现

## 三、核心技术要点与注意事项

### 3.1 ⚡ 元↔万元自动识别与转换（最高优先级）

**问题描述**：同一份预算PDF内，表1~表8的单位为"元"，表9的单位为"万元"，编制说明正文的单位为"万元"。如果不做正确的单位识别和转换，所有跨表、表文勾稽规则都会产生海量误报。

**实现方案**：
```
1. 解析每张表的表头行，识别"单位：元"/"单位:万元"/"单位：亿元"
2. 在取值时自动转换到统一基准单位（万元）
3. 万元转换规则：
   - "元" → ÷ 10000
   - "万元" → × 1
   - "亿元" → × 10000
4. 容差设定：
   - 表内勾稽（同单位）：±0.01元
   - 表文勾稽（跨单位）：±100元（即 ±0.01万元）
```

### 3.2 空白单元格 = 0 的处理

**问题描述**：PDF表格中，部分无数据的单元格显示为空白（如三公表中"公务用车"列），解析后可能为 `None`/`NaN`/空字符串。

**处理方案**：在所有数值运算前，空白单元格统一视为 `0`。

### 3.3 部门 vs 单位的规则分支

**实现方案**：
```yaml
rule_scope:
  department_only:  # 仅部门预算适用
    - B-006 (下属单位覆盖性声明)
    - B-010 (项目经费情况说明)
  unit_only:  # 仅单位预算适用
    - (暂无)
  common:  # 部门+单位通用
    - B-001 ~ B-005, B-007 ~ B-009, B-011 ~ B-016
```

**判定依据**：封面首行文字：
- 包含"区级**部门**预算" → 启用 department_only + common
- 包含"区级**单位**预算" → 启用 unit_only + common

### 3.4 同比说明的柔性处理

**决算 vs 预算差异**：
- 决算中的同比（"年初预算 vs 决算数"）需要**硬性**要求原因说明
- 预算中的同比（"本年预算 vs 上年预算"）需要**柔性**处理：
  - 出现"与上年持平" → 不要求额外原因说明 ✅
  - 出现"增加/减少 N 万元" → **必须**紧跟原因说明文字
  - 仅出现数值变动但无"持平/增加/减少"任何说明 → 报 `疑似` 缺失

### 3.5 编制说明中的科目名称匹配

**问题描述**：编制说明中使用的科目名称为自然语言格式（如 `"一般公共服务支出-政府办公厅（室）及相关机构事务-行政运行"`），而表中的名称可能有微小差异（如全角半角括号、换行截断等）。

**匹配策略**：
```
1. 预处理：统一全角→半角，去除多余空格
2. 匹配模式：取编制说明中"类-款-项"的最后一个层级名称（如"行政运行"）
3. 在表中按编码查找，再用名称做二次校验
4. 容许轻微差异（如"（室）"vs"(室)"）
```

## 四、与决算规则的继承关系

| 决算规则 | 预算对应 | 处理方式 |
|---------|---------|---------|
| D-001 收入支出决算总表 | B-001 总表平衡 | ✅ 直接继承，结构一致 |
| D-002 收入决算表 | B-008 编制说明验证 | ⚠️ 预算无独立的"收入情况说明"章节，改为编制说明验证 |
| D-003 支出决算表 | B-007 支出总表行级汇总 | ✅ 直接继承 |

## 六、样本PDF实测发现的潜在问题

在对这两份样本PDF做初步审查时，发现了以下**值得关注的点**（非一定为错误，但规则需要能处理）：

### 6.1 机关运行经费的微小差异
- 说明文字：`"机关运行经费预算为213.28万元"`
- 表8公用经费合计：`2,132,777.54元` = `213.2778万元`
- 差值：`0.0022万元 = 22元`
- **结论**：在容差范围内（±100元），不应报错。说明数字做了四舍五入处理。

### 6.2 编制说明科目金额的精度损失
- 说明写 `"行政运行"科目1,202.95万元`
- 表中 `12,029,501.84元` = `1,202.950184万元`
- 说明做了保留两位小数的四舍五入
- **结论**：容差设为 ±0.01万元（±100元）可以覆盖此类情况

### 6.3 三公表中公务用车列为空白
- 表9中"公务用车购置及运行费"区域无数字
- 说明里明确写"公务用车购置及运行费0万元"
- **结论**：解析引擎需要将空白/None统一处理为0

### 6.4 部门预算立项依据年份引用
- 项目经费说明引用了 `"《区委、区政府2024年重点工作任务书》"`
- 这里的"2024年"**不应**被年份一致性检查所拦截
- **结论**：年份规则需要对引用性内容（书名号、引号内）设置白名单

### 6.5 国有资产截止日期
- 说明写 `"截至2024年8月31日"`
- 这个日期是资产清查截止日，**不应**被误判为年份错误
- **结论**："截至"关键词后跟的日期应加入白名单
