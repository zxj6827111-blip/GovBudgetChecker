# 类款项名称一致性规则说明

## 适用范围

- 预算场景：`BUD-109`
- 决算场景：`V33-227`

以上两条规则都用于检查“表格中的功能分类类/款/项名称”与“对应情况说明中的类/款/项名称”是否一致。

## 判定基准

- 以表格为准：
  - 预算以 `BUD_T5` 为判定基准
  - 决算以 `T5` 为判定基准
- 情况说明中的类/款/项名称必须与表格口径一致。
- 若表格与说明名称不一致，则输出名称不一致问题。

## 比对逻辑

1. 优先按功能分类编码比对同一条目。
2. 编码存在时，直接比较该编码对应的表格名称与说明名称。
3. 编码缺失时，仅在可唯一映射到单一条目时才执行名称比对。
4. 无法唯一映射、证据不足或存在 OCR 风险时，降级为人工复核，避免误报。

## 结构化输出字段

规则命中后，会在 `location` 中补充以下字段：

- `expected_name`：表格中的标准名称
- `actual_name`：说明中的实际名称
- `code_level`：编码层级，如“类”“款”“项”
- `source_of_truth`：判定基准表，预算为 `BUD_T5`，决算为 `T5`

## 导出与前端展示

- CSV / JSON / PDF 标注导出已透出上述结构化字段。
- 前端问题卡片与问题列表已支持：
  - 五档严重级别展示
  - 类款项名称一致性分类
  - `expected_name` / `actual_name` / `code_level` / `source_of_truth` 展示

## 示例

- 表格名称：`卫生健康支出`
- 说明名称：`医疗卫生与计划生育支出`

上述情况会被识别为名称口径不一致，并以表格名称作为期望值输出。
