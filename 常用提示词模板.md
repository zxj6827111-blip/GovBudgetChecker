# 常用提示词模板

## 说明
下面这些提示词都可以直接复制给 Codex 使用。

如果根目录已经放了 `AGENTS.md`，这些提示词默认都会继承其中的预决算审校规则。

## 一、单份材料检查
### 1. 单份预算检查
```text
检查这份预算公开材料，按 AGENTS.md 规则执行，输出：
1. 审校结论
2. 问题清单
3. 待人工复核
4. 下一步建议
```

### 2. 单份决算检查
```text
检查这份决算公开材料，按 AGENTS.md 规则执行，重点检查：
1. 收入支出决算总表勾稽
2. 三公经费支出决算
3. 空表说明
4. 文字与数字一致性
```

### 3. 只输出问题
```text
检查这份材料，按 AGENTS.md 规则执行，只输出问题清单，不要展开写背景说明。
```

## 二、多份并行检查
### 4. 多份并行审校
```text
并行检查这几份预决算材料，按 AGENTS.md 规则执行。
分别输出：
1. 每份材料的问题清单
2. 每份材料的待人工复核项
3. 一个总汇总表，按严重级别排序
```

### 5. 多部门批量检查
```text
并行检查这些部门的预算公开材料，重点核查九张表、三公经费、绩效目标和勾稽关系。
最后输出一个汇总表，字段包括：
单位、年度、问题数量、critical数量、high数量、是否存在缺表、是否存在三公问题、是否建议人工复核。
```

### 6. 一次检查预算和决算
```text
并行检查这批材料，其中包含预算和决算文件。请先自动识别材料类型，再分别按 AGENTS.md 的预算规则和决算规则执行。
```

## 三、重点专项检查
### 7. 只看九张表
```text
检查这份预算公开材料，只重点看九张表或同类核心表是否齐全、是否错页、是否有重复表、表名和页码是否对应。
```

### 8. 只看三公经费
```text
检查这份材料中的三公经费部分，重点看：
1. 是否有三公经费表
2. 合计是否等于三项之和
3. 是否写明同比变化原因
4. 文本和表格金额是否一致
```

### 9. 只看绩效目标
```text
检查这份预算公开材料中的绩效目标内容，重点看：
1. 是否存在绩效目标表或绩效目标说明
2. 是否有部门整体绩效目标或重点项目绩效目标
3. 项目支出说明是否包含立项依据、测算依据、绩效目标、资金安排
```

### 10. 只看决算勾稽
```text
检查这份决算公开材料，只重点核查决算勾稽关系，尤其是：
1. 支出列总计 = 结余分配 + 年末结转和结余 + 本年支出合计
2. 基本支出 = 人员经费 + 公用经费
3. 三公经费决算合计与分项之和是否一致
```

## 四、结构化输出
### 11. 输出 JSON
```text
检查这份材料，按 AGENTS.md 规则执行，并按 JSON 输出结果，字段尽量包含：
rule_id、title、severity、page、bbox、table_or_section、snippet、expected、actual、difference、reason、suggestion、confidence。
```

### 12. 输出表格摘要
```text
检查这份材料，按表格形式输出摘要，列包括：
规则编号、问题标题、严重级别、页码、证据片段、建议。
```

### 13. 只输出高风险问题
```text
检查这份材料，只输出 critical 和 high 级别问题，并单独列出待人工复核项。
```

## 五、链接检查
### 14. 检查公开链接
```text
检查这个公开链接对应的预决算材料，按 AGENTS.md 规则执行，并输出结构化问题清单：
[把链接贴在这里]
```

### 15. 检查网页和附件是否对应
```text
检查这个公开页面及其附件，重点看：
1. 页面标题、附件标题、正文标题是否一致
2. 页面说明与 PDF 内容是否一致
3. 是否存在漏挂附件、错挂附件或附件名称不清晰的问题
[把链接贴在这里]
```

## 六、对比与复核
### 16. 对比两年材料
```text
对比检查这两份材料，重点看：
1. 同比口径是否一致
2. 三公经费变化是否写明原因
3. 项目支出变化是否合理
4. 是否疑似引用了上年度数据
```

### 17. 复核别人给出的结论
```text
下面是已有的审校结论，请按 AGENTS.md 规则帮我复核，指出哪些结论证据充分，哪些结论证据不足，哪些可能误报：
[把已有结论贴在这里]
```

### 18. 复核单条问题
```text
请只复核这一条问题是否成立，按 AGENTS.md 规则判断，并说明：
1. 证据是否充分
2. 是否可能是单位换算或四舍五入问题
3. 是否应降级为待人工复核
[把问题和证据贴在这里]
```

## 七、适合汇报的模板
### 19. 领导汇报版
```text
检查这批材料后，请输出一版适合汇报的结果：
1. 总体结论
2. 高风险问题
3. 共性问题
4. 需要人工复核的问题
5. 整改建议
```

### 20. 台账版
```text
检查这批材料，并生成适合做审校台账的结果，字段包括：
单位、年度、材料类型、问题标题、严重级别、页码、证据片段、整改建议、是否需人工复核。
```

## 八、最省事模板
如果你懒得每次想怎么说，直接复制这句：

```text
按 AGENTS.md 规则检查当前材料，自动识别是预算还是决算，并行检查结构完整性、核心表、必备说明、勾稽关系、三公经费、绩效目标、文字与数字一致性、同比与口径差异，输出问题清单和待人工复核项。
```
