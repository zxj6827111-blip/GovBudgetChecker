# 快速使用说明

## 1. 适用场景
这套用法适合在新电脑上直接使用 Codex 检查：

- 预算公开材料
- 决算公开材料
- 本地 PDF
- 公开链接
- 多份材料并行审校

## 2. 最简使用方式
把以下文件放到同一个工作区根目录：

- `AGENTS.md`
- 你要检查的 PDF 文件

然后用 Codex 打开这个文件夹。

之后你直接发一句话即可，例如：

```text
检查这份预算公开材料，按 AGENTS.md 规则输出问题清单。
```

## 3. 新电脑推荐目录
推荐像下面这样放：

```text
BudgetCheck/
├─ AGENTS.md
├─ 快速使用说明.md
├─ 常用提示词模板.md
├─ 某单位2025年预算公开.pdf
├─ 某单位2024年决算公开.pdf
└─ samples/
```

## 4. 如何发起检查
### 方法一：检查本地 PDF
把 PDF 放进当前工作区后，对 Codex 说：

```text
检查这个 PDF，按 AGENTS.md 规则执行。
```

如果有多份文件：

```text
并行检查当前目录下这几份 PDF，分别输出问题清单和总汇总。
```

### 方法二：检查公开链接
直接把链接发给 Codex：

```text
检查这个决算公开链接，按 AGENTS.md 规则执行，并输出结构化问题清单：
https://example.com/xxx
```

### 方法三：检查截图或粘贴文字
如果你暂时没有 PDF，也可以直接发：

- 截图
- 复制出来的说明文字
- 表格截图
- 表格文本

例如：

```text
检查这段决算说明文字是否存在口径或勾稽问题，按 AGENTS.md 规则执行。
```

## 5. 默认会检查什么
如果你没有额外说明，Codex 默认会检查：

- 材料类型识别
- 核心表是否齐全
- 九张表或决算核心表
- 必备说明章节
- 三公经费
- 绩效目标
- 表内表间勾稽
- 文字与数字一致性
- 同比和口径差异
- OCR 风险和待人工复核项

## 6. 默认输出什么
如果你没有特别要求，默认输出：

1. 审校结论
2. 问题清单
3. 待人工复核
4. 下一步建议

问题清单会尽量包含：

- 规则编号
- 问题标题
- 严重级别
- 页码
- bbox
- 命中片段
- 期望值
- 实际值
- 差额
- 建议
- 置信度

## 7. 最常用的 6 句指令
### 单份预算检查
```text
检查这份预算公开材料，按 AGENTS.md 规则输出问题清单。
```

### 单份决算检查
```text
检查这份决算公开材料，重点看勾稽关系、三公经费和空表说明。
```

### 多份并行检查
```text
并行检查这几份预决算材料，分别输出每份的问题清单、待人工复核项和一个总汇总表。
```

### 只看高风险问题
```text
检查这份材料，只输出 critical 和 high 级别问题。
```

### 输出为 JSON
```text
检查这份材料，按 AGENTS.md 的 JSON 模板输出结果。
```

### 对比两年差异
```text
对比检查这两份材料，重点看同比口径、项目变化和三公经费变化原因。
```

## 8. 使用建议
- 如果是多份材料，文件名里尽量带年份和单位名。
- 如果是扫描件，最好提醒 Codex“注意 OCR 风险”。
- 如果你只关心某一项，直接说清楚，例如“三公经费”“绩效目标”“空表说明”“决算勾稽”。
- 如果你要拿去汇报，建议要求输出 JSON 或表格化清单。

## 9. 一句话原则
只要根目录有 `AGENTS.md`，你以后直接说“检查这个预算/决算文件”即可，不用每次重写整套提示词。
