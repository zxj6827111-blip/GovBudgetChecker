# 政府预决算检查系统源代码分析报告

## 1. 软件整体功能定位

政府预决算检查系统是一个智能化的预决算文档分析工具，旨在提高政府预决算审查的效率和准确性。系统采用双模式检测机制，结合传统规则检查和AI智能分析，能够全面识别预决算文档中的合规性问题。

**核心价值**：
- 提高预决算审查效率，减少人工工作量
- 增强审查准确性，降低人为错误
- 支持多种文档格式和复杂结构
- 提供可视化的分析结果和友好的用户界面

## 2. 目标用户群体

- **政府财政部门**：负责预决算审查的工作人员
- **审计机构**：对政府预决算进行审计的专业人员
- **预决算编制单位**：自行检查预决算文档合规性的部门
- **监督机构**：对政府预决算执行情况进行监督的单位

## 4. 核心业务流程

### 4.1 文件上传与预处理

1. 用户通过前端界面上传PDF文件或提供URL
2. 后端验证文件格式和大小
3. 生成唯一job_id，保存文件
4. 返回job_id给前端，用于后续分析请求

### 4.2 双模式分析流程

1. **规则加载与分离**：
   - 加载配置的规则集
   - 将规则分为AI规则和引擎规则

2. **并行分析执行**：
   - 启动AI分析任务，使用大语言模型检测问题
   - 同时启动规则引擎任务，基于预定义规则检测问题
   - 使用asyncio实现真正的并行处理
   - 设置超时机制（120秒），确保系统稳定性

3. **结果合并与冲突检测**：
   - 合并AI分析结果和规则检测结果
   - 检测两种模式结果的冲突
   - 验证相同问题的检测一致性
   - 生成合并后的结果摘要

4. **结果存储与返回**：
   - 将结果保存到状态文件
   - 通过轮询机制返回结果给前端

### 4.3 结果展示与交互

1. 前端轮询获取分析状态和进度
2. 实时更新进度条和状态信息
3. 分析完成后，展示合并后的结果
4. 支持多种视图切换（标签页、列表、卡片）
5. 提供详细的问题信息查看

## 6. 技术亮点与创新点

1. **双模式融合技术**：
   - 同时运行AI和规则引擎，充分发挥各自优势
   - 智能合并结果，处理冲突和不一致
   - 提供置信度评估，增强结果可靠性

2. **多AI提供商支持**：
   - 支持多种AI提供商，包括OpenAI兼容接口、智谱AI、豆包AI

   - 负载均衡，优化资源利用

3. **现代化前端设计**：
   - 响应式设计，适配各种设备
   - 多种结果展示视图，满足不同用户需求
   - 实时进度可视化，提升用户体验

4. **高效并行处理**：
   - 使用asyncio实现真正的并行分析
   - 超时机制确保系统稳定性
   - 异常处理和降级策略，增强系统鲁棒性

5. **可扩展的规则系统**：
   - 支持AI规则和引擎规则分离
