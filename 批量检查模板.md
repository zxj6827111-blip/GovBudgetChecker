# 批量检查模板

## 使用说明
这份模板适合以下场景：

- 一次检查多份预算 PDF
- 一次检查多份决算 PDF
- 一次检查预算和决算混合材料
- 一次检查多个公开链接
- 做问题汇总台账
- 先批量筛查，再对高风险材料复核

建议原则：

- `5-10` 份材料为一批，最稳
- `20` 份以上先做清单识别，再分批检查
- 扫描件较多时，额外强调 OCR 风险
- 汇总时优先输出 `critical` 和 `high`

## 一、开始前的推荐目录
建议把文件这样放：

```text
BatchCheck/
├─ AGENTS.md
├─ 快速使用说明.md
├─ 常用提示词模板.md
├─ 批量检查模板.md
├─ 预算/
├─ 决算/
└─ links.txt
```

文件名尽量带：

- 年度
- 单位名
- 材料类型

例如：

- `2025_某单位_预算公开.pdf`
- `2024_某单位_决算公开.pdf`

## 二、批量任务最推荐流程
推荐按 3 步走：

1. 先识别文件清单
2. 再分批并行检查
3. 最后统一汇总高风险问题和台账

## 三、可直接复制的模板

### 1. 先识别当前目录材料
适合材料很多、文件名不规范时先做摸底。

```text
先扫描当前工作区里的全部材料，识别每份文件的：
1. 文件名
2. 单位名称
3. 年度
4. 材料类型（预算公开/决算公开）
5. 是否疑似扫描件

请先输出一个清单，暂时不要深入分析。
```

### 2. 批量并行检查当前目录全部 PDF
适合 5 到 15 份材料直接开跑。

```text
并行检查当前工作区内全部 PDF，按 AGENTS.md 规则执行。
请自动识别每份材料是预算还是决算，并分别检查：
1. 结构完整性
2. 核心表是否齐全
3. 必备说明章节
4. 表内表间勾稽
5. 三公经费
6. 绩效目标
7. 文字与数字一致性
8. 同比与口径差异

请输出：
1. 每份材料的问题清单
2. 每份材料的待人工复核项
3. 一个总汇总表
4. 一个只包含 critical 和 high 问题的汇总清单
```

### 3. 分批并行检查
适合 20 份以上材料。

```text
先按当前目录材料清单，把全部文件分成每批 5 到 10 份。
然后分批并行检查，每批按 AGENTS.md 规则执行。

每批输出：
1. 本批每份材料的问题清单
2. 本批每份材料的待人工复核项

全部批次完成后，再输出：
1. 总汇总表
2. 高风险问题汇总
3. 建议优先复核的材料名单
```

### 4. 只检查预算公开材料
```text
只检查当前工作区中的预算公开材料，按 AGENTS.md 规则执行。
重点检查：
1. 九张表或同类核心表
2. 预算编制说明
3. 三公经费预算说明
4. 绩效目标
5. 预算表与说明之间的金额一致性

输出每份材料的问题清单和汇总表。
```

### 5. 只检查决算公开材料
```text
只检查当前工作区中的决算公开材料，按 AGENTS.md 规则执行。
重点检查：
1. 收入支出决算总表勾稽
2. 基本支出和项目支出说明
3. 三公经费支出决算
4. 空表说明
5. 同比变化原因

输出每份材料的问题清单和汇总表。
```

### 6. 批量检查多个公开链接
适合你有一组网页链接时。

```text
按当前链接清单批量检查公开材料，按 AGENTS.md 规则执行。
请对每个链接分别完成：
1. 页面与附件识别
2. 材料类型判断
3. 结构完整性检查
4. 核心表和说明检查
5. 勾稽和口径检查

最后输出：
1. 每个链接对应的问题清单
2. 一个总汇总表
3. 一个高风险链接清单
```

### 7. 批量检查并生成台账
适合做内部跟踪表。

```text
并行检查这批材料，按 AGENTS.md 规则执行。
最后生成适合做审校台账的结果，字段包括：
1. 单位
2. 年度
3. 材料类型
4. 问题标题
5. 严重级别
6. 页码
7. 证据片段
8. 整改建议
9. 是否需人工复核
```

### 8. 只输出高风险问题
适合先筛一轮重点。

```text
并行检查这批材料，按 AGENTS.md 规则执行。
只输出以下内容：
1. critical 问题
2. high 问题
3. 待人工复核项

并按单位和严重级别汇总排序。
```

### 9. 扫描件专用模板
适合大量扫描版 PDF。

```text
并行检查这批扫描件材料，按 AGENTS.md 规则执行。
请特别注意：
1. OCR 风险
2. 页码识别偏差
3. 表格跨页断裂
4. 单位识别错误

证据不足的不要直接判错，统一转为待人工复核。
最后输出问题清单、待人工复核项和高风险汇总表。
```

### 10. 预算与决算混合批次模板
```text
并行检查这批混合材料，先自动识别每份是预算公开还是决算公开，再按 AGENTS.md 对应规则执行。
最后按两类分别汇总：
1. 预算类汇总
2. 决算类汇总
3. 全部高风险问题总表
```

### 11. 批量复核整改后版本
适合第二轮检查。

```text
这是整改后的第二轮材料。请重新批量检查，并和上一轮结论相比，输出：
1. 已整改完成的问题
2. 仍然存在的问题
3. 新出现的问题
4. 仍需人工复核的问题
```

### 12. 批量对比两年材料
适合同单位跨年比对。

```text
对比检查这批单位的两年材料，重点看：
1. 同比口径是否一致
2. 三公经费变化是否写明原因
3. 项目支出变化是否异常
4. 是否疑似沿用上年表述或数字

请分别输出每个单位的对比结论和总汇总表。
```

## 四、推荐输出汇总字段
批量任务里，建议固定要求输出以下汇总字段：

- 单位
- 年度
- 材料类型
- 问题总数
- critical 数量
- high 数量
- medium 数量
- low 数量
- manual_review 数量
- 是否缺核心表
- 是否存在三公问题
- 是否存在绩效目标问题
- 是否存在重大勾稽错误
- 是否建议优先复核

## 五、最省事的批量模板
如果你每次都不想改，直接复制这句：

```text
并行检查当前工作区内全部预决算材料，按 AGENTS.md 规则执行。请先识别每份材料的单位、年度和类型，再分批并行检查结构完整性、核心表、必备说明、勾稽关系、三公经费、绩效目标、文字与数字一致性、同比与口径差异。输出：
1. 每份材料的问题清单
2. 每份材料的待人工复核项
3. 一个总汇总表
4. 一个只包含 critical 和 high 问题的汇总清单
5. 一个建议优先复核的材料名单
```

## 六、实战建议
- 材料少时，直接整批并行检查。
- 材料多时，先识别清单，再分批跑。
- 扫描件多时，一定强调 OCR 风险。
- 如果你要做正式台账，让 Codex 输出 JSON 或表格摘要。
- 如果你要快速抓重点，先只看 `critical`、`high` 和 `manual_review`。
